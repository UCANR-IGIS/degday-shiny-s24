R Methods: Managing your API Key

General Best Practices

  • Create an app token or service account
  • Provision your token / key with the minimum number of permissions required
  • Take advantage of limitations provided by the service
  • IP filtering
  • expiration date
  • usage limits


Protecting Your Key from Escape

  • do not enter your token in your code
## Bad idea:
my_token <- "414c045492a32hhd9994xxxxd8a64df"
  • do not check-in files with your token into version control (i.e, GitHub)
  • options for bringing your token into memory:
  • environment variable: (good choice)
my_token <- Sys.getenv("MY_CENSUS_KEY")
  • external file:
my_token <- readLines("~/My Tokens/cimis_token.txt", n = 1)
  • OS credential store

Create a POSIXct object from scratch

Tip:

Days begin at midnight (00:00:00) and end at 23:59:59.

st_patricks_noon_pdt_dt <- make_datetime(year = 2024, month = 3, day = 17, 
                                      hour = 12, min = 0, sec = 0,
                                      tz = "America/Los_Angeles")

st_patricks_noon_pdt_dt
## [1] "2024-03-17 12:00:00 PDT"
